# BGV Platform — IIS Deployment Guide (Windows Server)

> **Audience**: Sysadmin / Dev deploying on Windows Server 2019/2022 or Windows 10/11 with IIS  
> **Architecture**: IIS acts as a reverse proxy for the FastAPI backend (Uvicorn) and serves the built React frontend as static files.

---

## Deployment Architecture

```
Client Browser (port 80 / 443)
         │
         ▼
    ┌──────────────────────────────────────────┐
    │         IIS Website (bgv.yourdomain.com) │
    │                                          │
    │  /          → Static files (React build) │
    │  /api/*     → Reverse Proxy ──────────── │──► Uvicorn @ localhost:8000
    │  /auth/*    → Reverse Proxy ──────────── │──► Uvicorn @ localhost:8000
    │  /ws/*      → WebSocket Proxy ────────── │──► Uvicorn @ localhost:8000
    └──────────────────────────────────────────┘
         │
         ▼
    ┌─────────────────────┐
    │  Windows Services   │
    │  • Uvicorn (NSSM)   │──► PostgreSQL @ 5432
    │  • Redis (Memurai)  │
    │  • Ollama (NSSM)    │
    │  • PostgreSQL svc   │
    └─────────────────────┘
```

## 2. Install & Enable IIS with Required Modules

### 2.1 Enable IIS via PowerShell (Windows Server)

```powershell
# Install IIS with management tools
Install-WindowsFeature -Name Web-Server -IncludeManagementTools

# Enable required IIS sub-features
Install-WindowsFeature -Name `
  Web-Common-Http, `
  Web-Static-Content, `
  Web-Default-Doc, `
  Web-Http-Errors, `
  Web-Http-Redirect, `
  Web-Http-Logging, `
  Web-Request-Monitor, `
  Web-Stat-Compression, `
  Web-Filtering, `
  Web-IP-Security, `
  Web-Url-Auth, `
  Web-Windows-Auth

# Verify IIS is running
Get-Service -Name W3SVC
```

### 2.2 Enable IIS via Windows Features (Windows 10/11)

```powershell
# Enable IIS and sub-features on desktop Windows
Enable-WindowsOptionalFeature -Online -FeatureName `
  IIS-WebServerRole, `
  IIS-WebServer, `
  IIS-CommonHttpFeatures, `
  IIS-StaticContent, `
  IIS-DefaultDocument, `
  IIS-HttpErrors, `
  IIS-HttpRedirect, `
  IIS-ApplicationDevelopment, `
  IIS-NetFxExtensibility45, `
  IIS-ASPNET45, `
  IIS-ISAPIExtensions, `
  IIS-ISAPIFilter, `
  IIS-HttpCompressionStatic, `
  IIS-Security, `
  IIS-RequestFiltering, `
  IIS-ManagementConsole `
  -All
```

### 2.3 Install URL Rewrite Module

```powershell
# Download URL Rewrite 2.1 from Microsoft
$urlRewriteUrl = "https://download.microsoft.com/download/1/2/8/128E2E22-C1B9-44A4-BE2A-5859ED1D4592/rewrite_amd64_en-US.msi"
$installer = "$env:TEMP\urlrewrite.msi"
Invoke-WebRequest -Uri $urlRewriteUrl -OutFile $installer
Start-Process msiexec.exe -ArgumentList "/i `"$installer`" /quiet /norestart" -Wait
Remove-Item $installer

# Verify — check IIS Manager for URL Rewrite module
```

### 2.4 Install Application Request Routing (ARR 3.0)

> [!IMPORTANT]
> ARR enables IIS to act as a reverse proxy to the Python backend. **This is required.**

```powershell
# Download ARR 3.0 installer
$arrUrl = "https://download.microsoft.com/download/E/9/8/E9849D6A-020E-47E4-9FD0-A023E99B54EB/requestRouter_amd64.msi"
$installer = "$env:TEMP\arr.msi"
Invoke-WebRequest -Uri $arrUrl -OutFile $installer
Start-Process msiexec.exe -ArgumentList "/i `"$installer`" /quiet /norestart" -Wait
Remove-Item $installer
```

### 2.5 Enable ARR Proxy

```powershell
# Enable the proxy feature in ARR (REQUIRED for reverse proxy to work)
# This must be done via IIS Manager OR via appcmd:
& "$env:SystemRoot\system32\inetsrv\appcmd.exe" set config `
  -section:system.webServer/proxy `
  /enabled:"True" `
  /commit:apphost
```

> [!NOTE]
> Alternatively, open **IIS Manager → Server Level → Application Request Routing Cache → Server Proxy Settings → Enable proxy** checkbox.

## 3. Install NSSM (Service Manager)

NSSM (Non-Sucking Service Manager) wraps any executable as a Windows Service — used to keep Uvicorn and Ollama running persistently.

```powershell
# Download NSSM
$nssmUrl = "https://nssm.cc/release/nssm-2.24.zip"
$nssmZip = "$env:TEMP\nssm.zip"
Invoke-WebRequest -Uri $nssmUrl -OutFile $nssmZip
Expand-Archive -Path $nssmZip -DestinationPath "$env:TEMP\nssm" -Force

# Copy the 64-bit binary to a permanent location
New-Item -ItemType Directory -Force -Path "C:\tools\nssm"
Copy-Item "$env:TEMP\nssm\nssm-2.24\win64\nssm.exe" -Destination "C:\tools\nssm\nssm.exe"

# Add to PATH
[System.Environment]::SetEnvironmentVariable(
  "Path",
  $env:Path + ";C:\tools\nssm",
  [System.EnvironmentVariableTarget]::Machine
)

# Reload PATH in current session
$env:Path += ";C:\tools\nssm"

# Verify
nssm version
```

## 4. Setup PostgreSQL as a Windows Service

PostgreSQL installs as a Windows Service automatically. Verify and configure:

```powershell
# Check service name (typically postgresql-x64-15 or postgresql-x64-16)
Get-Service -Name "postgresql*"

# Set to Automatic startup
Set-Service -Name "postgresql-x64-15" -StartupType Automatic

# Start the service
Start-Service -Name "postgresql-x64-15"

# Create the database and user for production (if not done)
psql -U postgres -c "CREATE USER bgv_user WITH PASSWORD 'YOUR_STRONG_PROD_PASSWORD';"
psql -U postgres -c "CREATE DATABASE bgv_db OWNER bgv_user;"
psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE bgv_db TO bgv_user;"

# Verify connectivity
psql -U bgv_user -d bgv_db -h localhost -c "SELECT version();"
```

> [!CAUTION]
> Use a **strong password** in production. Replace `YOUR_STRONG_PROD_PASSWORD` throughout this guide.

## 6. Setup Ollama as a Windows Service

```powershell
# 1. Ensure Ollama is installed: https://ollama.com/download/windows
# 2. Pull required model (do this BEFORE registering as service)
ollama pull llama3.1:latest
# OR lightweight:
ollama pull phi3:mini

# 3. Stop the Ollama tray app if running (right-click tray icon → Quit)

# 4. Register Ollama as a Windows Service via NSSM
nssm install OllamaService "C:\Users\$env:USERNAME\AppData\Local\Programs\Ollama\ollama.exe"
nssm set OllamaService AppParameters "serve"
nssm set OllamaService AppDirectory "C:\Users\$env:USERNAME\AppData\Local\Programs\Ollama"
nssm set OllamaService DisplayName "Ollama AI Service"
nssm set OllamaService Description "Ollama local LLM inference server for BGV Platform"
nssm set OllamaService Start SERVICE_AUTO_START

# Set environment variable for Ollama host binding
nssm set OllamaService AppEnvironmentExtra "OLLAMA_HOST=0.0.0.0:11434"

# 5. Start the service
nssm start OllamaService

# 6. Verify (allow 10-15 seconds to start)
Start-Sleep -Seconds 15
curl http://localhost:11434/api/tags
```

## 8. Register the Backend as a Windows Service

Use NSSM to keep Uvicorn running as a Windows Service that auto-starts with the server.

```powershell
# 1. Install the BGV Backend service
nssm install BGVBackend "C:\bgv\backend\.venv\Scripts\python.exe"

# 2. Set the startup arguments (production: no --reload, 2 workers)
nssm set BGVBackend AppParameters `
  "-m uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 2"

# 3. Set working directory (CRITICAL — app imports are relative to this)
nssm set BGVBackend AppDirectory "C:\bgv\backend"

# 4. Set environment variables
nssm set BGVBackend AppEnvironmentExtra `
  "KMP_DUPLICATE_LIB_OK=TRUE" `
  "PROTOCOL_BUFFERS_PYTHON_IMPLEMENTATION=python"

# 5. Configure log files
nssm set BGVBackend AppStdout "C:\bgv\logs\backend_stdout.log"
nssm set BGVBackend AppStderr "C:\bgv\logs\backend_stderr.log"
nssm set BGVBackend AppRotateFiles 1
nssm set BGVBackend AppRotateBytes 10485760

# 6. Set service metadata
nssm set BGVBackend DisplayName "BGV Platform Backend"
nssm set BGVBackend Description "FastAPI/Uvicorn backend for BGV Background Verification Platform"
nssm set BGVBackend Start SERVICE_AUTO_START

# 7. Set service dependencies (start after PostgreSQL and Redis)
nssm set BGVBackend DependOnService "postgresql-x64-15 Memurai"

# 8. Start the service
nssm start BGVBackend

# 9. Verify
Start-Sleep -Seconds 10
Get-Service -Name BGVBackend
curl http://localhost:8000/api/v1/health
```

> [!NOTE]
> Adjust `postgresql-x64-15` to your actual PostgreSQL service name (check with `Get-Service postgresql*`).

### Useful NSSM Service Commands

```powershell
# Check service status
nssm status BGVBackend

# Stop / Start / Restart
nssm stop BGVBackend
nssm start BGVBackend
nssm restart BGVBackend

# Edit service configuration
nssm edit BGVBackend

# Remove service
nssm remove BGVBackend confirm
```

## 10. Configure IIS Site

### 10.1 Create the IIS Website

```powershell
# Remove the Default Web Site if it's on port 80 (optional)
Remove-WebSite -Name "Default Web Site" -ErrorAction SilentlyContinue

# Create the BGV website pointing to the frontend build
New-WebSite `
  -Name "BGVPlatform" `
  -Port 80 `
  -HostHeader "bgv.yourdomain.com" `
  -PhysicalPath "C:\bgv\frontend" `
  -ApplicationPool "BGVAppPool" `
  -Force

# If deploying locally without a domain, use:
# New-WebSite -Name "BGVPlatform" -Port 80 -PhysicalPath "C:\bgv\frontend" -ApplicationPool "BGVAppPool" -Force
```

### 10.2 Create a Dedicated Application Pool

```powershell
# Create Application Pool
New-WebAppPool -Name "BGVAppPool"

# Configure it — No managed code (we're serving static files + proxying)
Set-ItemProperty `
  -Path "IIS:\AppPools\BGVAppPool" `
  -Name "managedRuntimeVersion" `
  -Value ""

# Set identity (NetworkService is fine; LocalSystem for more permissions)
Set-ItemProperty `
  -Path "IIS:\AppPools\BGVAppPool" `
  -Name "processModel.userName" `
  -Value "NetworkService"

Set-ItemProperty `
  -Path "IIS:\AppPools\BGVAppPool" `
  -Name "processModel.identityType" `
  -Value 2

# Start the pool
Start-WebAppPool -Name "BGVAppPool"
```

## 11. Create web.config Files

### 11.1 Root `web.config` — Frontend + API Reverse Proxy

Create `C:\bgv\frontend\web.config`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>

    <!-- ── Static file serving ───────────────────────────── -->
    <staticContent>
      <mimeMap fileExtension=".webmanifest" mimeType="application/manifest+json" />
      <mimeMap fileExtension=".woff"  mimeType="font/woff" />
      <mimeMap fileExtension=".woff2" mimeType="font/woff2" />
      <mimeMap fileExtension=".svg"   mimeType="image/svg+xml" />
    </staticContent>

    <!-- ── Default document ──────────────────────────────── -->
    <defaultDocument>
      <files>
        <clear />
        <add value="index.html" />
      </files>
    </defaultDocument>

    <!-- ── HTTP Compression ──────────────────────────────── -->
    <httpCompression>
      <dynamicTypes>
        <add mimeType="application/json" enabled="true" />
        <add mimeType="text/javascript"  enabled="true" />
      </dynamicTypes>
    </httpCompression>

    <!-- ── URL Rewrite Rules ─────────────────────────────── -->
    <rewrite>
      <rules>

        <!-- Rule 1: Proxy /api/* → Uvicorn backend -->
        <rule name="ReverseProxy-API" stopProcessing="true">
          <match url="^api/(.*)" />
          <conditions>
            <add input="{HTTP_HOST}" pattern=".*" />
          </conditions>
          <action type="Rewrite"
                  url="http://localhost:8000/api/{R:1}"
                  appendQueryString="true" />
        </rule>

        <!-- Rule 2: Proxy /auth/* → Uvicorn backend (OAuth callbacks) -->
        <rule name="ReverseProxy-Auth" stopProcessing="true">
          <match url="^auth/(.*)" />
          <action type="Rewrite"
                  url="http://localhost:8000/auth/{R:1}"
                  appendQueryString="true" />
        </rule>

        <!-- Rule 3: WebSocket proxy /api/v1/ws/* → Uvicorn -->
        <rule name="ReverseProxy-WebSocket" stopProcessing="true">
          <match url="^api/v1/ws/(.*)" />
          <conditions>
            <add input="{HTTP_UPGRADE}" pattern="websocket" flags="None" />
          </conditions>
          <action type="Rewrite"
                  url="http://localhost:8000/api/v1/ws/{R:1}"
                  appendQueryString="true" />
        </rule>

        <!-- Rule 4: SPA fallback — serve index.html for React Router -->
        <!-- Must be LAST rule -->
        <rule name="SPA-Fallback" stopProcessing="true">
          <match url=".*" />
          <conditions logicalGrouping="MatchAll">
            <!-- Skip if file exists (JS/CSS/images) -->
            <add input="{REQUEST_FILENAME}" matchType="IsFile"      negate="true" />
            <add input="{REQUEST_FILENAME}" matchType="IsDirectory"  negate="true" />
            <!-- Skip API/auth routes (already handled above) -->
            <add input="{REQUEST_URI}"      pattern="^/api"         negate="true" />
            <add input="{REQUEST_URI}"      pattern="^/auth"        negate="true" />
          </conditions>
          <action type="Rewrite" url="/index.html" />
        </rule>

      </rules>

      <!-- Preserve original host header when proxying -->
      <outboundRules>
        <rule name="ReverseProxyOutboundRule" preCondition="ResponseIsHtml1">
          <match filterByTags="A, Form, Img" pattern="^http://localhost:8000/(.*)" />
          <action type="Rewrite" value="/{R:1}" />
        </rule>
        <preConditions>
          <preCondition name="ResponseIsHtml1">
            <add input="{RESPONSE_CONTENT_TYPE}" pattern="^text/html" />
          </preCondition>
        </preConditions>
      </outboundRules>
    </rewrite>

    <!-- ── Security Headers ──────────────────────────────── -->
    <httpProtocol>
      <customHeaders>
        <remove name="X-Powered-By" />
        <add name="X-Content-Type-Options"  value="nosniff" />
        <add name="X-Frame-Options"         value="SAMEORIGIN" />
        <add name="X-XSS-Protection"        value="1; mode=block" />
        <add name="Referrer-Policy"         value="strict-origin-when-cross-origin" />
      </customHeaders>
    </httpProtocol>

    <!-- ── Request Filtering (upload size) ───────────────── -->
    <security>
      <requestFiltering>
        <!-- Allow up to 52 MB uploads (matches MAX_UPLOAD_SIZE_MB=50 + overhead) -->
        <requestLimits maxAllowedContentLength="54525952" />
      </requestFiltering>
    </security>

    <!-- ── Error Pages ───────────────────────────────────── -->
    <httpErrors errorMode="Custom" existingResponse="PassThrough" />

  </system.webServer>
</configuration>
```

## 13. Setup HTTPS / SSL

### Option A: Self-Signed Certificate (Internal / Testing)

```powershell
# Create self-signed certificate
$cert = New-SelfSignedCertificate `
  -DnsName "bgv.yourdomain.com" `
  -CertStoreLocation "cert:\LocalMachine\My" `
  -NotAfter (Get-Date).AddYears(2)

# Bind HTTPS to IIS site
New-WebBinding `
  -Name "BGVPlatform" `
  -Protocol "https" `
  -Port 443 `
  -HostHeader "bgv.yourdomain.com"

# Assign the certificate to the HTTPS binding
$thumbprint = $cert.Thumbprint
& "$env:SystemRoot\system32\netsh.exe" http add sslcert `
  hostnameport="bgv.yourdomain.com:443" `
  certhash="$thumbprint" `
  certstorename="MY" `
  appid="{4dc3e181-e14b-4a21-b022-59fc669b0914}"
```

### Option B: Let's Encrypt (Production — Free, trusted)

```powershell
# Install win-acme (ACME client for Windows/IIS):
# https://www.win-acme.com/

# Download and extract win-acme
Invoke-WebRequest `
  -Uri "https://github.com/win-acme/win-acme/releases/latest/download/win-acme.v2.x.x.x86.pluggable.zip" `
  -OutFile "$env:TEMP\wacs.zip"
Expand-Archive "$env:TEMP\wacs.zip" -DestinationPath "C:\tools\wacs"

# Run the interactive ACME client
C:\tools\wacs\wacs.exe
# Follow prompts:
#   N (new certificate)
#   1 (IIS site binding)
#   Select BGVPlatform
#   Complete domain verification
```

### Redirect HTTP → HTTPS

Add this rule **above** all other rules in `web.config`:

```xml
<!-- Add inside <rules> before all other rules -->
<rule name="HTTP to HTTPS Redirect" stopProcessing="true">
  <match url="(.*)" />
  <conditions>
    <add input="{HTTPS}" pattern="off" ignoreCase="true" />
  </conditions>
  <action type="Redirect"
          url="https://{HTTP_HOST}/{R:0}"
          redirectType="Permanent" />
</rule>
```

## 14. Firewall Rules

```powershell
# Allow HTTP (port 80)
New-NetFirewallRule `
  -DisplayName "BGV Platform HTTP" `
  -Direction Inbound `
  -Protocol TCP `
  -LocalPort 80 `
  -Action Allow

New-NetFirewallRule `
  -DisplayName "BGV Platform HTTPS" `
  -Direction Inbound `
  -Protocol TCP `
  -LocalPort 443 `
  -Action Allow

# Block direct access to backend port from outside (only IIS should reach it)
New-NetFirewallRule `
  -DisplayName "Block Direct Backend Access" `
  -Direction Inbound `
  -Protocol TCP `
  -LocalPort 8000 `
  -RemoteAddress "!127.0.0.1" `
  -Action Block

# Block direct Redis access from outside
New-NetFirewallRule `
  -DisplayName "Block Direct Redis Access" `
  -Direction Inbound `
  -Protocol TCP `
  -LocalPort 6379 `
  -RemoteAddress "!127.0.0.1" `
  -Action Block

New-NetFirewallRule `
  -DisplayName "Block Direct Ollama Access" `
  -Direction Inbound `
  -Protocol TCP `
  -LocalPort 11434 `
  -RemoteAddress "!127.0.0.1" `
  -Action Block
```

## 16. Troubleshooting

### 16.1 502 Bad Gateway — Backend Not Reachable

**Symptom:** Browser shows `502 - Bad Gateway` when accessing `/api/*`

**Diagnosis:**
```powershell
# 1. Is Uvicorn actually running?
Get-Service BGVBackend
curl http://localhost:8000/api/v1/health

# 2. Is ARR proxy enabled?
& "$env:SystemRoot\system32\inetsrv\appcmd.exe" list config -section:system.webServer/proxy

# 3. Check backend logs
Get-Content "C:\bgv\logs\backend_stderr.log" -Tail 20
```

**Fix:**
```powershell
# Restart the backend service
nssm restart BGVBackend

# Re-enable ARR proxy
& "$env:SystemRoot\system32\inetsrv\appcmd.exe" set config `
  -section:system.webServer/proxy /enabled:"True" /commit:apphost

iisreset /restart
```

---

### 16.2 404 on Page Refresh (React Router)

**Symptom:** Navigating to `/dashboard` directly returns 404

**Fix:** The SPA-Fallback rule in `web.config` must be present. Verify:

```powershell
Get-Content "C:\bgv\frontend\web.config" | Select-String "SPA-Fallback"
# Should output the rule name
```

If missing, re-add the SPA-Fallback rewrite rule from Section 11.1.

---

### 16.3 WebSocket Connection Fails (Live Updates Broken)

**Symptom:** Real-time document processing updates don't appear

**Checklist:**
```powershell
# 1. WebSocket IIS feature installed?
Get-WindowsFeature Web-WebSockets    # Should show Installed

# 2. webSocket enabled in web.config?
Get-Content "C:\bgv\frontend\web.config" | Select-String "webSocket"

# 3. ARR WebSocket pass-through enabled
& "$env:SystemRoot\system32\inetsrv\appcmd.exe" set config `
  -section:system.webServer/proxy /disableProxyResponseBuffer:"True" /commit:apphost
```

### 16.4 CORS Errors in Browser

**Symptom:** Browser console shows `Access-Control-Allow-Origin` errors

**Fix:** Update `backend/.env`:
```ini
# Include your production domain exactly
CORS_ORIGINS=https://bgv.yourdomain.com
```

Then restart the backend:
```powershell
nssm restart BGVBackend
```

---

### 16.5 Google OAuth Redirect URI Mismatch

**Symptom:** `Error 400: redirect_uri_mismatch` from Google

**Fix:**
1. Go to [Google Cloud Console → Credentials](https://console.cloud.google.com/apis/credentials)
2. Edit your OAuth 2.0 Client ID
3. Add to **Authorized redirect URIs**: `https://bgv.yourdomain.com/auth/callback`
4. Update `backend/.env`: `GOOGLE_REDIRECT_URI=https://bgv.yourdomain.com/auth/callback`
5. Restart backend: `nssm restart BGVBackend`

### 16.6 File Upload Fails (413 Request Entity Too Large)

```powershell
  -section:requestFiltering `

```
